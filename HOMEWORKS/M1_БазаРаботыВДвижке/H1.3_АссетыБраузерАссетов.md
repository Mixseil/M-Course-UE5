## Структурирование проекта и работа с импортом
[Конспект того занятия](LESSONS\M1_БазаРаботыВДвижке\L1.3_АссетыБраузерАссетов.md)
### Задание 1: Создание структуры папок

Создайте в **Content Browser** следующую структуру папок в корне проекта:

1. `Content/Maps` — для уровней
2. `Content/Meshes` — для 3D-моделей
3. `Content/Materials` — для материалов и их экземпляров
4. `Content/Textures` — для текстур
5. `Content/Blueprints` — для Blueprint-ассетов
6. `Content/Audio` — для звука
7. `Content/UI` — для интерфейса
8. `Content/Fab` — для контента из Fab

**Проверка:** все папки видны в **Content Browser** слева в дереве папок, переходы между ними работают без ошибок.

> **ВАЖНО:** Папки создаются **только через Content Browser** (ПКМ → New Folder), а не через проводник Windows.


### Задание 2: Импорт простого ассета вручную

Найдите или создайте три простых файла для импорта:
- один `.fbx` или `.obj` файл (3D-модель, даже простой куб/сфера);
- одну текстуру (`.png` или `.tga`, размер хотя бы 512×512);
- один звуковой файл (`.wav` или `.mp3`).

Если нет готовых файлов, можете:
- скачать простые модели с сайтов вроде Sketchfab (в формате `.fbx`);
- найти текстуру на Poly Haven или аналогичном ресурсе;
- взять любой короткий звуковой файл.

Импортируйте их в соответствующие папки:
1. 3D-модель → `Content/Meshes`
2. Текстуру → `Content/Textures`
3. Звук → `Content/Audio`

**Способ импорта:** перетащите файлы из проводника прямо в нужную папку **Content Browser** (Drag and Drop).

**Проверка:** все три ассета лежат в правильных папках, видны в **Content Browser**, имеют логичные названия (не "NewAsset1", а что-то вроде "SM_Cube", "T_Stone_BaseColor", "SFX_Click").

> **СОВЕТ:** если при импорте меша открылось окно с параметрами, просто нажмите **Import** или **Import All** - базовые настройки по умолчанию обычно работают нормально. Но в условиях комплексной 3D модели можете вспомнить про настройку из урока - **CombineMeshes** 


### Задание 3: Открытие импортированных ассетов

Для каждого импортированного ассета:

1. **Двойной клик** по ассету в **Content Browser** и посмотрите, в каком редакторе он открылся:
   - Меш должен открыться в **Static Mesh Editor**
   - Текстура в **Texture Viewer**
   - Звук в **Sound Cue Editor** или **Sound Wave Player**

2. Закройте редактор ассета (вкладка или окно).


### Задание 4: Работа с типами ассетов и фильтрами

1. Откройте папку `Content/Meshes` в **Content Browser**.

2. В верхней части браузера найдите фильтры по типам ассетов (это может быть выпадающее меню или набор кнопок вроде "All", "Static Mesh", "Skeletal Mesh" и т.д.).

3. Включите фильтр так, чтобы показывались **только Static Mesh**.

4. Убедитесь, что видны только меши, которые вы импортировали.

5. Сотрите фильтр, чтобы вернуться в режим "All".

> Фильтры очень полезны, когда в одной папке лежит много разного контента. Привыкайте к ним сразу.


### Задание 5: Навигация и переименование

1. В папке `Content/Meshes` найдите импортированный меш.

2. Нажмите на нём **ПКМ** → выберите **Rename** (или просто нажмите **F2** после выделения).

3. Переименуйте так, чтобы название было понятным и следовало конвенции `SM_` (для Static Mesh):
   - например, если это куб, то `SM_Cube`
   - если это столб, то `SM_Pillar`
   - если это камень, то `SM_Rock`

4. Нажмите **Enter** для сохранения.

5. Сделайте то же самое для текстуры (приставка `T_`) и звука (приставка `SFX_` или `S_`).

**Проверка:** все три ассета переименованы логично и читаемо.

> **ПОЛЕЗНО ЗНАТЬ:** в UE есть негласные условные обозначения префиксов, вот небольшая часть из них:
> - `SM_` для Static Mesh
> - `SK_` для Skeletal Mesh
> - `T_` для Texture
> - `M_` для Material
> - `MI_` для Material Instance
> - `BP_` для Blueprint
> - `SFX_` для звука
> Тут важно отметить, что не обязательно следовать конкретно этим правилам в вашем проекте, вы можете разработать и другие правила нейминга, тут важно им следовать в принципе, и чтобы они были одни на проект.

Это не обязательно, но когда проект растёт, такие префиксы экономят кучу времени на поиск.

### Задание 7: Самопроверка и организация

Откройте **Content Browser** и посмотрите на свой проект с высоты птицы:

1. Папки созданы и логично организованы?
2. В каждой папке лежит правильный тип контента?
3. Названия ассетов понятны и следуют конвенции?
4. Все импортированные ассеты видны и открываются без ошибок?

Если что-то не так, исправьте это **сейчас**, пока проект ещё маленький.

## На что обратить внимание

- **Не удаляйте файлы через проводник.** Если хотите удалить ассет, делайте это через **Content Browser** (ПКМ → Delete).
- **Сохраняйте проект часто.** После импорта и переименования нажмите **Ctrl+S** или **File** → **Save All**.
- **Если вы переместили или переименовали ассет через Content Browser, это безопасно.** Движок сам обновит все ссылки.
- **Если вы переместили ассет через проводник, это опасно.** Движок может потерять ссылки на него.


## Полезные горячие клавиши

- **Ctrl+Space** — открыть/закрыть **Content Browser**
- **Ctrl+S** — сохранить текущий ассет или уровень
- **Ctrl+Shift+S** — сохранить всё (Save All)
- **F2** — переименовать выбранный ассет в **Content Browser**
- **Del** — удалить выбранный ассет
- **Ctrl+F** — открыть поиск в **Content Browser**
