## Подключение проекта к GitHub и тестирование

### Задание 1: Установка компонентов

1. Установите **Git** с [git-scm.com](https://git-scm.com) (выберите `main` как ветка)
2. Установите **Git LFS** с [git-lfs.com](https://git-lfs.com) и выполните `git lfs install` в командной строке
3. Установите **GitHub Desktop** с [desktop.github.com](https://desktop.github.com)
4. Авторизуйтесь в GitHub Desktop со своим аккаунтом GitHub


### Задание 2: Подключение проекта к Git

1. Откройте GitHub Desktop
2. Нажмите **File → Add Local Repository**
3. Выберите папку вашего UE5 проекта
4. Нажмите **Create Repository**

**Проверка:** GitHub Desktop показывает ваш проект с файлами.


### Задание 3: Создание .gitattributes

1. В корне проекта создайте/измените файл `.gitattributes` (рядом с `.uproject`)
2. Добавьте строки для LFS:
```
*.uasset filter=lfs diff=lfs merge=lfs -text
*.umap filter=lfs diff=lfs merge=lfs -text
*.png filter=lfs diff=lfs merge=lfs -text
*.tga filter=lfs diff=lfs merge=lfs -text
*.fbx filter=lfs diff=lfs merge=lfs -text
*.wav filter=lfs diff=lfs merge=lfs -text
```

**Проверка:** файл сохранён в корне проекта.


### Задание 4: Публикация на GitHub

1. В GitHub Desktop нажмите **Publish repository**
2. Введите название проекта и поставьте **Keep this code private**
3. Нажмите **Publish Repository**

Дождитесь завершения загрузки.

**Проверка:** репозиторий появился на github.com.


### Задание 5: Первый коммит и пуш

1. Откройте UE5 проект и сделайте небольшое изменение
2. Сохраните (**Ctrl+S**)
3. В GitHub Desktop напишите сообщение коммита в поле **Summary**
4. Нажмите **Commit to main**
5. Нажмите **Push origin**

**Проверка:** коммит появился на GitHub.com.


### Задание 6: Создание веток

1. В GitHub Desktop нажмите **Current Branch → New Branch**
2. Создайте ветку `develop`
3. Опубликуйте её (**Publish branch**)

**Проверка:** в GitHub Desktop видны две ветки: `main` и `develop`.


### Задание 7: Работа в разных ветках

1. Убедитесь, что находитесь в ветке `develop`
2. Откройте UE5 и сделайте изменение
3. Сохраните и сделайте коммит
4. Нажмите **Push origin**
5. Переключитесь на `main` через **Current Branch**
6. Вернитесь на `develop`

**Проверка:** вы поняли, как переключаться между ветками.


### Задание 8: Клонирование репозитория

1. Создайте новую папку на диске
2. В GitHub Desktop нажмите **File → Clone Repository**
3. Выберите ваш репозиторий
4. Клонируйте в новую папку

**Проверка:** проект успешно загружен, все файлы на месте.

## Что сдавать на проверку - если хочется сдать и подписаны на бусти

1. Скриншот GitHub Desktop с историей коммитов и ветками
2. Ссылка на репозиторий на GitHub
3. Краткое описание: какой проект подключили, вызвала ли какая-то сложность?
